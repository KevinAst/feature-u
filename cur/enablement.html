Feature Enablement

Feature Enablement

Each feature has a Feature.enabled boolean property that determines whether it is enabled or not. This indicator is typically based on a dynamic expression.

This allows packaged code to be dynamically enabled/disabled at run-time, and is useful in a number of different situations. For example:

  • some features may require a license upgrade

  • other features may only be used for diagnostic purposes, and are disabled by default

If need be you can use the App object to determine if a feature is present or not (see: Checking Feature Dependencies (via App)).
